virtual Date QuantLib::InflationTermStructure::baseDate ( ) const [pure virtual]

minimum (base) date

Important in inflation since it starts before nominal reference date. Changes depending whether index is interpolated or not. When interpolated the base date is just observation lag before nominal. When not interpolated it is the beginning of the relevant period (hence it is easy to create interpolated fixings from a not-interpolated curve because interpolation, usually, of fixings is forward looking).

Implemented in QuantLib::InterpolatedYoYCapFloorTermPriceSurface< Interpolator2D, Interpolator1D >, QuantLib::CPICapFloorTermPriceSurface, QuantLib::PiecewiseYoYInflationCurve< Interpolator, Bootstrap, Traits >, QuantLib::PiecewiseZeroInflationCurve< Interpolator, Bootstrap, Traits >, QuantLib::InterpolatedYoYInflationCurve< Interpolator >, and QuantLib::InterpolatedZeroInflationCurve< Interpolator >.

Referenced by checkRange(), QuantLib::MultiplicativePriceSeasonality::correctYoYRate(), QuantLib::MultiplicativePriceSeasonality::correctZeroRate(), and QuantLib::MultiplicativePriceSeasonality::isConsistent().

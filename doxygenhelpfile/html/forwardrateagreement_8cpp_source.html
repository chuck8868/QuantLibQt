00078     // In theory, no need to implement this for a FRA (could directly
00079     // supply a forwardValue). For the sake of keeping a consistent
00080     // framework, we adhere to the concept of the forward contract as
00081     // defined in the base class, with an underlying having a
00082     // spotPrice (in this case, a loan or deposit with an NPV). Thus,

00084     Real ForwardRateAgreement::spotValue() const {
00085         calculate();

00087                forwardRate().compoundFactor(valueDate_, maturityDate_) *
